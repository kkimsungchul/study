# 메가존
메가존은 1998년 웹 에이전시/호스팅 사업으로 IT 산업에 첫 발을 내딛었습니다. 
글로벌 IT 기업과 파트너십 체결 및 유수의 국내 엔터프라이즈 고객사의 사이트 구축 및 운영/호스팅을 통한 IT 시장 저변의 확대를 시작하였습니다. 
메가존은 IT 산업에 안주하지 않고, 광고/영상으로의 사업 확장을 통해 국내 산업 전반의 흐름을 선도하고 있으며, 
개발자/서비스기획자/디자이너/광고AE/아트디렉터 등 다양한 직무의 직원들이 어우러진 회사입니다. 
메가존은 직원들의 성장을 위해 직무 관련 교육, 도서 지원 등을 아끼지 않으며, 
다양한 복지 제도(버디 프로그램 운영, 복지비 연 100만원 지원 등)를 통해 직원이 행복한 회사를 지향하고자 합니다.

 

◆ 메가존.디지털 소개

메가존.디지털은 메가존의 여러 사업부 중, IT전반을 담당하는 사업부로 고객의 비즈니스 환경 변화에 맞는 디지털 트랜스포메이션 기술, 
최신 IT 기술 융합 서비스, 비즈니스 혁신을 위한 디지털 서비스를 담당하고 있습니다.


# 자기소개
	안녕하십니까 메가존 개발자 채용에 지원한 김성철입니다.
	현재 KT본사에 파견근무를 하고있으며 개발자들이 사용하는 Devops 툴인 JIRA, Jenkins , Gitlab등을 연계하여 
	하나의 사이트에서 관리 할 수 있도록 해주는 Dev-space 포탈의 개발 및 운영 업무를 맡고 있습니다.

	이전 회사에서는 AD와 연동을 통한 통합계정관리솔루션을 개발 및 구축 하였으며,
	AWS의 EC2에 통합계정관리솔루션을 별도로 구축하여 외부에서 시연자료로 사용할 수 있도록 했습니다.
	
	이전회사에서 프로젝트를 진행하며 키워온 서비스 연계 능력과 개발경험을 토대로
	개발 및 운영 업무에 참여하여 능력을 충분히 발휘하고 메가존과 함께 성장하고 싶어 지원하였습니다.

# 자기소개에서 파생되어 나올 수 있는 질문
	1. 타 시스템과의 연계를 해보셧다고 했는데 어떻게 하셨나요?
		Devops 툴들간의 연계는 해당툴에서 제공하는 API를 통해서 하였습니다.
		Jira , Jenkins, Gitlab에서는 API Document 를 제공해주고 있어서 API Document 를 보고서 개발하였습니다.
		커스터마이징이 필요한 작업의 경우에는 클라이언트와 서버가 주고받는 데이터를 분석하여 API를 개발하였습니다.

	2. Jenkins 로 자동 배포나 빌드 테스트는 해보셨나요?
		프리스타일 프로젝트와 파이프라인을 통한 빌드 배포를 진행했습니다.
		현재 담당하고 있는 프로젝트에서 젠킨스로 빌드, 배포를 하고 있습니다.
	
# 예상 질문 리스트 - 이력서
	1. 불필요한 리소스 제거에 소요되는 시간 감소를 위한 데이터 동기화 기능 개발
		현재 맡은 포털 사이트에서 프로젝트를 생성 시 Sparrow라는 코드 취약점 점검 프로젝트도 같이 생성하고 있습니다.
		Sparrow는 단독으로 작동하지 않고 깃랩, 젠킨스와 연계되어 작동하는 방식으로 사용자가 더이상 깃랩 프로젝트를 사용하지 않을 경우
		Sparrow를 사용하지 않지만 Sparrow는 깃랩 프로젝트가 삭제되었는지 알수 있는 방법이 없어 일일히 제거 하고 있었습니다.
		깃랩과 젠킨스, 스패로우의 데이터를 연계하여, 깃랩의 프로젝트가 비활성화 또는 삭제가 되었을 경우 젠킨스와 Sparrow가 비활성화 될수 있도록 데이터 동기화 기능을 개발하였습니다.				
		
	2. Keycloak 암호화 모듈 개발
		Keycloak 는 오픈소스로 제공되는 SSO 솔루션입니다.
		구글에 검색을 하면 이미 다른 개발자가 만들어 놓은 암호화 모듈이 있지만, 
		해당 모듈은 Keycloak에서 제공하는 DB를 사용하는 경우에만 작동하고 LDAP를 연동하여 사용하는 상태에서는 작동하지 않았습니다.
		다른 개발자가 개발해 놓은 모듈을 토대로 LDAP에서 작동할 수 있도록 새로 개발하였습니다.

		2-1. 왜LDAP에서 안됐나요?
			다른 개발자가 만들어 놓은 암호화 모듈은 복호화 하는 시점에서 문제가 발생하였습니다.
			Keycloak DB를 사용할 경우 사용자가 입력한 암호를 DB와 비교하기 전에 복호화를 하지만
			LDAP을 사용할 경우 사용자가 입력한 암호를 복호화 하기 전에 LDAP에 저장되어 있는 암호와 비교하여 틀리다고 나왔습니다.
			이부분을 파악하여 LDAP과 비교하기 전에 복호화 하는 모듈을 개발했습니다.

	3. 프로젝트 생성 소요 시간 감소를 위한 Jenkins 동시 생성 기능 개발
		포털에서 깃랩 프로젝트 생성 시 기본적인 파이프라인 스크립트를 포함하여 Jenkins job 도 같이 생성할수 있도록 개발하였습니다.

	
	4. 사내 접속시 SMS인증을 일 1회만 하도록 설계하여 비용 50% 감소
		이전 포털에서는 로그인시 매번 SMS로 2차인증을 하도록 설계되어있었습니다.
		한달에 65,000건의 문자발송을 기본적으로 할수있었지만 이용하는 사용자가 늘어나면서 매달 약 2만건의 문자를 추가로 발송하였습니다.
		사용자가 외부망이 아닌 내부망에서 로그인할 경우 SMS인증을 일 1회만 하도록 구현하여 SMS발송건수를 약 4만건 으로 줄였습니다.
	
	5. Portal CI/CD 도입을 통해 빌드/배포에 소요되는 시간 83% 감소
		포털사이트는 개발 당시 Jenkins 를 사용하여 빌드 및 배포를 하지 않고 개발자가 일일히 서버로 배포 파일을 넘겨 배포하는 방식으로 진행을 해왔습니다.
		Web 2대 , Was 6대를 배포하고 서비스 상태 확인에 약 2시간이 소요되었습니다.
		현재 사용중인 모든 Web과 Was를 Jenkins로 빌드 배포를 할수 있게 파이프라인을 구축하여 약 20분정도로 줄였습니다.
	
	6. 프로세스 자동화로 인한 일일 업무 소요시간 일 평균 30분이상 감소
		포털사이트에 연동된 DevOps툴들에 할당된 메모리와 용량 체크를 아침에 관리자가 출근하면 일일히 서버에 들어가서 점검하였습니다.
		각각의 서버에 들어가서 메모리와 CPU사용율, 잔여 디스크용량을 확인하여 메일발송을 하도록 하는데 매일 30분정도의 시간이 소요되었습니다.
		이부분을 각각 리눅스에 크론탭과 스프링의 스케줄러를 사용하여 자동화를 해놓아 해당 업무에 소요되는 시간을 없앴습니다. 
	
	7. 인사시스템 연동 , 로그인 History DB테이블 설계
		IDMS 시스템과 연동하여 퇴사 및 입사시 데이터를 체크하는 부분을 개발하였습니다.
		또한 사용자가 로그인 할 경우 로그인 기록을 남기는 테이블 설계 및 기능을 개발하였습니다.

	8. 보안관제 업무 로그 분석 프로그램 개발로 업무 소요 시간 66%단축
		이전회사에서는 보안관제 서비스도 제공하고 있었습니다.
		매일 아침 출근 시 해당 서버에 접속하여 로그파일을 내려 받아 공격이 의심되는 내용들을 분석하여 공격이 발견될 경우 해당 내용을 정리하여 메일을 발송하였습니다.
		다만 공격내용을 일일이 직원이 검색하여 찾아야 했습니다.
		자주 들어오는 공격패턴을 정리하여 로그파일내에 해당 내용을 탐색하고 정리해주는 분석프로그램을 개발하여 1시간정도의 작업시간을 20분으로 줄였습니다.
	
	9. 통합계정관리 솔루션 개발 및 구축 - 고도화 진행 완료 후 기능 추가에 소요되는 시간 약 50% 이상 감소
		기존 통합계정관리 솔루션의 매니저 프로그램은 Spring / JSP / Jquery 로 구현하였습니다.
		개발 당시 신입개발자여서 코드를 작성할때 응집도나 결합도를 고려하지 않고 개발하다보니 추후 추가적인 요구사항이나 다른 회사에 도입할 경우
		기존 코드를 다 갈아엎고 다시 개발해야됬습니다.
		이러한 문제점을 회사에 얘기하고 SpringBoot와 Thymeleaf로 고도화를 진행하여 기능 추가 및 변경에 소요되는 시간을 약 50% 이상 줄였습니다.
		
		9-1. 어째서 50%인가요?
			사용자 목록의 특정 사용자에게 SMS 발송 기능을 추가하면서 기존에 개발한 기능을 손대지 않고 추가만 하면 되었기에 50%가 감소되었습니다.
			
# 이직 사유 지원동기
	1. 이전 직장에서 이직한 이유는 무엇입니까? 
		KT의 Dev-space 포털의 개발 및 안정화, 환경 개선까지 모든 과정을 진행하여 사내 전 직원이 원활하게 사용 중입니다.
		저는 항상 사용자의 입장에서 서비스 개발하고 유지보수를 진행합니다. 그 결과, 2021년 서비스를 런칭해 사용자 약 1000명에서 시작하여 올해 4000명 이상으로 약 300% 이상의 사용자 증가를 이뤘습니다.
		위 경험을 통해 이룬 '고객 중심 개발 마인드'를 메가존의 서비스에 적용해보려 합니다.


	2. 전 직장에서 가장 크게 기여한 부분은 무엇인가요?
		Jenkins 서비스 연계 부분과 롤백 및 재동기화 개발 입니다.
		Jenkins 의 생성 수정 삭제 원격빌드를 사용자가 포털에서 간편하게 할 수 있도록 개발하였습니다.
		또한 하나의 포털에서 API를 통하여 각종 툴들을 관리하다보니 생성 및 수정작업에서 장애 발생 시 롤백 및 재동기화를 할수 있도록 정책을 정하고 개발하였습니다.


		2-1. 롤백정책은 어떻게 하였나요?
			롤백정책은 각종 툴에 API 를 통하여 생성 수정 삭제시 보내는 데이터 또는 수정하기전 데이터 를 DB에 임시저장 해놓았다가, 서비스 장애시 저장되어 있는 데이터를 토대로 원상복구 할 수 있도록 하였습니다.


	3. 이전직장에서 이해관계자들과의 관계는 어떠하셨나요?
		업무를 믿고 맡길수 있는 개발자였습니다.
		현재 맡은 업무에서 정기배포 이후 정상적인 서비스가 가동되는 것을 보고 퇴근하였는데, 밤 11시에 서비스 장애가 발생한 적이 있었습니다. 
		늦은시간이였고 업무시간에만 사용하는 서비스라서 다들 다음날 아침에 일찍 출근해서 서비스 장애에 대한 분석 및 정상화를 하고자 하였습니다.
		아침에 출근해서 바로 정상화가 될 수 없을 가능성도 있었기에 저는 장애발생을 인지한 시점에 회사로 출근하여 로그를 확인하고 다음날 아침에 바로 조치가 가능하도록 준비를 하였고, 
		준비한 내용을 토대로 아침일찍 다른 직원들이 출근하였을때 같이 작업을 진행하여 업무시간 전에 서비스를 정상화 하였습니다
		이러한 업무에 대한 책임감으로인하여 다른 직원들에게 업무에 대한 신뢰를 쌓았습니다.


		3-1. 무슨장애였나요?
			로그인 서비스에서 문자 발송이 안되는 오류였습니다.
			문자 발송시스템은 문자 발송 요청시간이 10분이상 차이가 나면 발송을 하지않도록 설정이 되어있었는데,
			문자발송을 요청하는 서버는 타임서버와의 연동이 되지 않아 시간이 조금씩 느려지고 있었습니다.
			배포를 하지않았어도 결국 문제가 발생 할 사안이였습니다.

# 자기 PR / 자신의 주요 경력을 간략하게 소개해주세요

		둘째, 능동적인 업무 자세
		현 관리 중인 서비스의 Linux 서버에 대한 리소스를 매일 확인하고 공유를 하는 업무가 있습니다. 해당 단순 업무에 대한 자동화를 제안하고 개발하여 워크로드 30분을 해소하였습니다.
		또한 현 서비스의 빌드 및 배포를 기존에는 수동으로 진행하였으나, Jenkins를 통한 CI/CD 파이프라인 구축을 통해 해당 워크로드 약 80% 단축에 성공하였습니다.
		이처럼 현 상황에 안주하지 않고, 더 나은 환경으로 개선하기 위해 고민하고 실행해왔습니다.

		위 경험을 통해 쌓아온 저의 기술 및 능동적 업무 자세는, '메가존의 서비스’에 도움이 될 것이라 자신합니다.

	2. 지원하는 직무를 성공적으로 수행하는데 도움이 될 기술이나 경험은 어떤 것이 있나요?
		서비스 연계능력과 다양한 툴 사용 경험입니다.
		현재 맡고 있는 업무에서 jenkins , gitlab , jira 툴을 사용하고 서비스 연계를 해본 경험과
		이전 회사에서 진행했던 EC2에 통합계정관리서비스 구축해본 경험이 있습니다.
		

	3. 지원한 직무에 가장 중요한 역량은 뭐라고 생각하시나요?
		개발 역량도 중요하지만 고객의 불편사항을 빠르게 파악해서 개선해 나갈수 있는 고객중심의 개발 마인드가 중요하다고 생각합니다.
	
	4. 문제 해결에 있어서 본인은 어떻게 해결했는지?
		현재 진행중인 프로젝트는 별도의 API문서가 없이 진행되고 있었습니다.
		스웨거를 사용하여 API를 확인하면 되지만 여러개의 서비스가 연계되어 운영중이기에 API하나만을 파악한다고 하여 전체적인 시스템의 흐름을 알수 없습니다.
		이러한 상태에서 장애가 발생할 경우 대처가 느릴수 있다는 문제점을 파악하고 confluence에 API문서를 작성 할 수 있도록 해놓았습니다.

# 장점 / 단점

	1. 당신의 장점은 무엇입니까
		책임과 끈기, 문제해결 능력입니다.
		현재 진행중인 업무에서 접속하려는 사용자들의 IP를 체크하여 접속을 허용 또는 차단할수 있는 기능을 개발해 달라는 요구사항이 있었습니다.
		로그인 서비스를 제공해주는 keycloak 툴에서는 IP 체크를 할수 있는 방법이 없었기에 해당 기능을 불가능으로 마무리 하려고 했지만,
		IP체크를 로그인 서비스를 제공하는 툴 직접 하지않고 외부 API를 호출하여서 체크하면 가능할것 같다는 생각이들어 로그인 모듈중간부분에 API 를 추가하여 IP 체크를 할 수 있도록 하였습니다.
				
	
	2. 당신의 단점은 무엇입니까
		끈기가 너무 강한 것도 단점이라고 생각합니다. 
		이전에 통합계정관리 서비스를 구축할때 사용자의 정보를 엑셀로 내려받는 기능을 개발한 적이있습니다.
		화면단에서 사용한 자바스크립트 라이브러리에서 엑셀파일을 바로 내려받을 수 있는 기능이 있기에 사용하였었는데
		데이터가 2만건이 넘어가면 다운로드가 안되었습니다.
		자바스크립트 라이브러리를 통해서 9만건이 되는 사용자 데이터를 다운로드 하기위해 3일간 라이브러리 파일을 이곳저곳 수정하여서 시도하였지만
		결국 실패하고 백엔드에서 데이터를 처리하여 엑셀다운로드 기능을 개발하여 반나절 만에 끝냈습니다.
		자바스크립트로 다운로드하는것에 대해 좀더 타협하여 2만건 이하는 자바스크립트로 그 이상은 백엔드에서 처리하도록 하였으면 시간소요가 이렇게 크지 않았을것입니다.

Q.Iaas, saas, paas 차이점 또는 설명 해주세요
	IaaS(Infrastructure-as-a-service) , 클라우드 서비스의 가장 기본적인 유형으로 OS부터 미들웨어까지 전부다 사용자가 관리해야하는 하며, 대표적으로 EC2가있습니다.
	PaaS (Platform as a Service), 소프트웨어를 개발할수 있는 플랫폼까지 제공해주는 유형, 코드만 개발해서 바로 배포하면 되는 수준으로, 대표적으로 Red Hat OpenShift
	SaaS (Software as a Service), 클라우드 기반의 소프트웨어 자체를 서비스로 제공해주는 유형, 인터넷만 되면 별도의 설치 없이 바로 사용가능하며 , 대표적으로 Google Drive , Office365, 노션
