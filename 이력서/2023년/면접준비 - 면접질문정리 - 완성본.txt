# 자기소개
	안녕하십니까 신세계아이앤씨 스파로스 CMP 개발직무에 지원한 김성철입니다.
	현재 KT본사에 파견근무를 하고있으며 개발자들이 사용하는 Devops 툴인 JIRA, Jenkins , Gitlab등을 연계하여 
	하나의 사이트에서 관리 할 수 있도록 해주는 D-space 포탈의 개발 및 운영 업무를 맡고 있습니다.

	이전 회사에서는 AD와 연동을 통한 통합계정관리솔루션을 개발 및 구축 하였으며,
	AWS의 EC2에 통합계정관리솔루션을 별도로 구축하여 외부에서 시연자료로 사용할 수 있도록 했습니다.
	
	이전회사에서 프로젝트를 진행하며 키워온 서비스 연계 능력과 클라우드 서비스를 사용해본 경험을 토대로 
	스파로스CMP 개발업무에 참여하여 능력을 충분히 발휘하고 신세계아이앤씨와 함께 성장하고 싶어 지원하였습니다.
	

# 자기소개에서 파생되어 나올 수 있는 질문
	1. 퍼블릭 클라우드 경험 유무 - 경험을 했다면 어떤 CSP에서 어떤 기능을 사용하였는지와 역할 
		AWS 의 EC2를 사용해봤습니다.
		EC2 서비스를 사용하여 통합계정관리 솔루션과 해당 솔루션에서 사용하는 데이터베이스를 구축하였습니다.
		또한 elastic IP 를 사용하여 외부에서 접속 할 수 있도록 하였습니다.

		1-1 클라우드 서비스에 대해 아는게 있나요?
			IaaS , 클라우드 서비스의 가장 기본적인 유형으로 OS부터 미들웨어까지 전부다 사용자가 관리해야하는 하며, 대표적으로 EC2가있습니다.
			PaaS , 소프트웨어를 개발할수 있는 플랫폼까지 제공해주는 유형, 코드만 개발해서 배로 배포하면 되는 수준으로, 대표적으로 Red Hat OpenShift
			SaaS , 클라우드 기반의 소프트웨어 자체를 서비스로 제공해주는 유형, 인터넷만 되면 별도의 설치 없이 바로 사용가능하며 , 대표적으로 Google Drive , Office365, 노션

			1-2. 오픈시프트에대해 말씀하셨는데 써보셨나요?
				오픈시프트와 오픈시프트의 오픈소스인 OKD는 경험해본 적이 없습니다.
				도커와 쿠버네티스를 제공하는 컨테이너 애플리케이션 플랫폼으로 Devops 환경까지 제공해주는 것으로 알고있습니다.
				
			

			1-3. 도커는 사용해보셨나요?
				테스트환경을 구축할 때 도커를 이용해서 컨테이너 환경을 구축해봤습니다

	2. 타 시스템과의 연계를 해보셧다고 했는데 어떻게 하셨나요?

		Devops 툴들간의 연계는 해당툴에서 제공하는 API를 통해서 하였습니다.
		Jira , Jenkins, Gitlab에서는 API Document 를 제공해주고 있어서 API Document 를 보고서 개발하였습니다.
		커스터마이징이 필요한 작업의 경우에는 버프스위트라는 도구를 사용하여 클라이언트와 서버가 주고받는 데이터를 분석하여 API를 개발하였습니다.


	3. Jenkins 로 자동 배포나 빌드 테스트는 해보셨나요?
		프리스타일 프로젝트와 파이프라인을 통한 빌드 배포를 진행했습니다.
		현재 담당하고 있는 프로젝트에서 젠킨스로 빌드, 배포를 하고 있습니다.
	
	4. CMP에 대해 아시는대로 말씀해보세요
		
		클라우드 서비스에 관심이 있어 공부하고 사용해보다가 CMP에 대해 알게되었습니다.
		CMP는 클라우드 매니지먼트 플랫폼으로 다양한 클라우드 시스템에서 제공해주는 기능들을 표준화 하여 사용자들이 편리하게 클라우드 서비스를 사용할 수 있도록 해준다고 되어있습니다.
		그래서 CMP를 서비스하는 기업들을 찾아보니 신세계아이앤씨와 SKT , 베스핀글로벌 등의 업체가 서비스 하는것을 확인하였습니다.
		SKT_CMP와 베스핀글로벌의 OpsNow는 비용관리에 중점을 둔 CMP로 VM생성에 대한 부분은 보이지 않았습니다.
		클라우드 매니지먼트 플랫폼이라고 한다면 VM을 생성하고 관리 할수있어야 하는데 그 부분이 명확하게 보이는 CMP는 없었습니다.
		신세계아이앤씨의 스파로스 CMP는 조사를 해보아도 뉴타닉스(Nutanix) 인프라 기반의 IaaS CMP라는 정보만을 접할수 있었습니다.
		뉴타닉스는 클라우드 서비스를 제공해 주는 기업이며, 뉴타닉스 기반이라고 한다면 VM 생성까지 한다고 추측했습니다.

		=================================================================================================================
		2022년 행정·공공기관 클라우드컴퓨팅 수요예보 자료에 따르면 현재 공공분야의 시스템은1만6000개 정도입니다.
		이 시스템들의 클라우드 이용률 13.3% 이지만 향후 클라우드 도입 예정은 28.9% 라고 합니다.
		또한 25년까지 공공기관의 정보시스템을 100% 클라우드로 전환할 계획 입니다.
		앞으로 더 많은 시스템들과 기관에서 클라우드 서비스를 사용할 때 스파로스CMP가 앞장 설 수 있도록 하기 위하여 지원하였습니다.

	
	5. 이전회사에서 사용해왔던 기술중 CMP에 도입할수 있는 부분이 있나요.
		CMP에서 클라우드 서비스들을 관리한다면 사용자들의 클라우드 서비스 접근 권한관리까지도 필요합니다.
		이전회사에서 IM을 개발 및 구축할 당시 사용자들을 각 시스템 별로 권한을 관리 할수 있는 IAM(Identity Access Management) 접근권한 관리 까지 같이 개발했었습니다.
		

		

		======================================================================================================
		CMP (Cloud Management Platform)는 클라우드를 통합 관리하는 플랫폼입니다.
		CMP는 다양한 클라우드 시스템이 자체적으로 고유하게 갖고 있던 기능들을 공통된 규격으로 서비스화하여 제공
			- 관리의 접근이 아닌 서비스 사용의 접근으로 시작
		CMP는 클라우드 서비스를 조회하고 신청하는 워크플로우를 일원화, 표준화하였습니다.

		CMP는 사용자 중심의 클라우드 플랫폼 운영이 가능
			- 사용자와 관리자의 플랫폼을 이원화함으로써 사용자가 주도적으로 자원을 생성하고 관리할 수 있는 환경을 제공
		CMP는 사용자와 개발자 모두에게 최고의 환경을 제공 
			- CMP는 사용자 플랫폼을 제공함으로써 사용 난이도를 낮추고 셀프서비스 환경을 제공함으로써 사용자의 권한을 증대시킴
			- 사용자에게 클라우드 환경 운영의 효용을 경험하기 위해서는 사용자의 자유로움이 보장

			※ 플랫폼 운영의 목적은 단순하게 기능을 붙이는 것이 아니라, 다양한 기능과 서비스들을 표준으로 관리하여 유연성과 효율성의 가치를 제공하는 것

	3. 클라우드의 동향 - 향후 퍼블릭의 우세 or 프라이빗의 우세 => 
		
		클라우드 시장은 앞으로 하이브리드 클라우드 쪽으로 더욱 더 성장할 것으로 보입니다.
		정부.공공기관의 클라우드 서비스 도입에 대한 자료예보에 따르면 국내 1만6천여개 시스템중에 13.3% 만 클라우드 서비스를 도입한 것으로 나오며, 향후 28.9%는 클라우드 서비스를 도입할 예정이라고 조사되었습니다.
		또한 2025년까지 정부의 정보시스템들을 100% 클라우드로 전환할 계획이 있다고 합니다.
		이와같이 퍼블릭 클라우드와 프라이빗 클라우드 둘다 성장할 것으로 보이며, 
		엔터프라이즈환경에서는 데이터 보안과 과금정책 정부규제 준수때문에 프라이빗 클라우드를 사용하지만, 퍼블릭 클라우드의 유연성이 필요하기때문에 하이브리드 클라우드쪽으로 가야된다고 생각합니다.



	4. DevOps / MSA / CI/CD 의 차이점 or 경험 ?
		
		4-1. DevOps 경험

			DevOps 툴은 Jira와 Confluence, Gitlab , Jenkins 를 사용했습니다.
			jira 와 Confluence 를 사용하여 개발 일정 및 개발내용을 협의하고 Gitlab을 사용하여 코드 관리를 하였습니다.
			빌드 및 배포는 Jenkins 를 사용했습니다.

		4-2. MSA
			MSA는 직접 사용해본적은 없지만 서비스가 커지면 저킬수록 모놀로틱 형태의 개발에서 발생할수 있는 문제를 보완해줄수 있습니다.
			문제점으로는 부분장애가 전체 서비스의 장애로 이어지는경우, 
			빌드, 테스트, 배포시간이 늘어나는 경우, 서비스를 부분적으로 scale-out 해야하는 경우등이 있습니다.
			MSA를 도입하면 전체 서비스 중단 없이 서비스별 배포가 가능하고, 특정 서비스의 확장성이 좋아집니다.
			또한 장애가 전체 서비스로 확장될 가능성도 적습니다.

			다만 서비스간 통신을 사용하기에 서비스간 호출 API가 많아지며
			테스트 / 트랜젝션의 복잡도가 증가합니다.
			또한 데이터가 여러 서비스에  걸쳐 분산되기 때문에 한번에 조회하기가 어렵습니다.
			
			scale-out 노드 증가
			scale-in 노드 감소
			scale-up 서버 사양 증가
			scale-down 서버 사양 감소
			
		4-3. CI/CD
			현재 맡은 프로젝트에서 Jenkins 사용하여 빌드 및 배포를 하고 있습니다.

# 회사에 대해 얼마나 아는가?


	1. 회사에서 하고 있는 사업에 대해 아는게 있는가?
		IT서비스와 유통부분으로 사업하는것으로 알고 있습니다.
		IT서비스는 크게 시스템 운영(SM) 및 시스템개발/통합(SI), 클라우드, 디지털 공간 서비스, AI 사업을 하고있는 것으로 알고있고
		유통부문은 IT관련 장비 및 솔루션 유통, 각종 디바이스 IT기기 유통 사업을 하는것으로 알고 있습니다.
		스파로스는 Cloud 및 AI 기반의 Digital Transformation 기술을 아우르는 리테일테크 통합 브랜드입니다.

	4. 왜 이 일을 하고 싶습니까?
		스파로스 CMP 개발업무를 통하여 고객중심의 클라우드 플랫폼 서비스 개발을 하고 싶어 지원하였습니다.
		2022년 행정·공공기관 클라우드컴퓨팅 수요예보 자료에 따르면 현재 공공분야의 정보 시스템은1만6000개 정도입니다.
		이 시스템들의 클라우드 이용률 13.3% 이지만 향후 클라우드 도입 예정은 28.9% 라고 합니다.
		또한 25년까지 공공기관의 정보시스템을 100% 클라우드로 전환할 계획 입니다.
		앞으로 더욱 더 많은 시스템들과 기관에서 클라우드 서비스로 전환을 할때 스파로스CMP와 함께하고 성장하고 싶어 지원했습니다.
		
	5. 이직할 회사를 고르는 본인의 가장 큰 기준은 무엇인지?
		현재에 안주하지않고 함께 꾸준히 성장 할 수 있는 회사를 선택합니다.
		선택하는 기준은 최신 트렌드에 맞는 기술들을 서비스하려고 하거나 서비스를 하고 있는가로 판단합니다.
			


# 이직 사유
	1. 이전 직장에서 이직한 이유는 무엇입니까? 
		서비스 연계 및 개발에 대한 역량을 발전시키고 싶고, 개인적으로 클라우드에 대해 관심이 있어서 스파로스 CMP 개발 직무에 지원하게 되었습니다.
		현재 직장에서는 각종 Devops툴들을 연계하여 사용할 수 있도록 하는 프로젝트를 진행했습니다.
		2차 고도화까지 끝나고 현재는 사소한 수정사항들만 개발하고 있으며, 추가적으로 기능들을 개발할 예정이 없이 운영 업무만이 남아있습니다.
		현재에 안주하지 않고 더욱더 발전을 하기 위하여 이직을 결심하였습니다.
				
		
	2. 전 직장에서 가장 크게 기여한 부분은 무엇인가요?
		Jenkins 서비스 연계 부분과 롤백 및 재동기화 개발 입니다.
		Jenkins 의 생성 수정 삭제 원격빌드를 사용자가 포털에서 간편하게 할 수 있도록 개발하였습니다.
		또한 하나의 포털에서 API를 통하여 각종 툴들을 관리하다보니 생성 및 수정작업에서 장애 발생 시 롤백 및 재동기화를 할수 있도록 정책을 정하고 개발하였습니다.


		2-1. 롤백정책은 어떻게 하였나요?
			롤백정책은 각종 툴에 API 를 통하여 생성 수정 삭제시 보내는 데이터 또는 수정하기전 데이터 를 DB에 임시저장 해놓았다가, 서비스 장애시 저장되어 있는 데이터를 토대로 원상복구 할 수 있도록 하였습니다.



	3. 이전직장에서 이해관계자들과의 관계는 어떠하셨나요?
		업무를 믿고 맡길수 있는 개발자였습니다.
		현재 맡은 업무에서 정기배포 이후 정상적인 서비스가 가동되는 것을 보고 퇴근하였는데, 밤 11시에 서비스 장애가 발생한 적이 있었습니다. 
		늦은시간이였고 업무시간에만 사용하는 서비스라서 다들 다음날 아침에 일찍 출근해서 서비스 장애에 대한 분석 및 정상화를 하고자 하였습니다.
		아침에 출근해서 바로 정상화가 될 수 없을 가능성도 있었기에 저는 장애발생을 인지한 시점에 회사로 출근하여 로그를 확인하고 다음날 아침에 바로 조치가 가능하도록 준비를 하였고, 
		준비한 내용을 토대로 아침일찍 다른 직원들이 출근하였을때 같이 작업을 진행하여 업무시간 전에 서비스를 정상화 하였습니다
		이러한 업무에 대한 책임감으로인하여 다른 직원들에게 업무에 대한 신뢰를 쌓았습니다.


		3-1. 무슨장애였나요?
			로그인 서비스에서 문자 발송이 안되는 오류였습니다.
			문자 발송시스템은 문자 발송 요청시간이 10분이상 차이가 나면 발송을 하지않도록 설정이 되어있었는데,
			문자발송을 요청하는 서버는 타임서버와의 연동이 되지 않아 시간이 조금씩 느려지고 있었습니다.
			배포를 하지않았어도 결국 문제가 발생 할 사안이였습니다.

# 자기 PR / 자신의 주요 경력을 간략하게 소개해주세요
	1. 당신의 직무 경험에 대해 말해주세요	
		현재 KT본사에 파견근무를 하고있으며 개발자들이 사용하는 개발툴인 JIRA, Jenkins , Gitlab등을 연계하여 
		하나의 사이트에서 관리 할 수 있도록 해주는 D-space 포탈 개발 및 운영 업무를 맡고 있습니다.
		이전 회사에서는 인사시스템과 데이터 연동을 통한 통합계정관리서비스를 개발 및 구축 하였습니다.

	2. 지원하는 직무를 성공적으로 수행하는데 도움이 될 기술이나 경험은 어떤 것이 있나요?
		서비스 연계능력과 클라우드 경험 입니다.
		현재 맡고 있는 업무에서 jenkins , gitlab , jira 와의 서비스 연계 경험과
		이전 회사에서 진행했던 EC2에 통합계정관리서비스 구축해본 경험이 있습니다.
		

	3. 지원한 직무에 가장 중요한 역량은 뭐라고 생각하시나요?
		지원한 직무가 CMP인 만큼 클라우드 서비스에 대한 역량과 클라우드 서비스와의 연계 능력이라고 생각합니다.
	
	4. 문제 해결에 있어서 본인은 어떻게 해결했는지?
		현재 맡고 있는 업무는 별도의 API 문서 없이 개발을 진행하고 있었습니다.
		여러 개발자가 같이 개발을 하고, API를 개발했던 개발자가 퇴사를 하고 나면 해당 API에서 오류가 발생하거나 수정해야 할 때 시간이 많이 소요된다는 문제를 알았습니다.
		긴급히 수정해야 할 경우에 API문서가 없어서 제대로 분석이 안된다면 장시간 서비스 장애로 이어질 수 있다는 것을 파악하고
		현재는 API 개발 시 confluence 툴에 API문서를 작성할 수 있도록 해놓았습니다.
		


# 장점 / 단점

	1. 당신의 장점은 무엇입니까
		책임과 끈기, 문제해결 능력입니다.
		현재 진행중인 업무에서 접속하려는 사용자들의 IP를 체크하여 접속을 허용 또는 차단할수 있는 기능을 개발해 달라는 요구사항이 있었습니다.
		로그인 서비스를 제공해주는 keycloak 툴에서는 IP 체크를 할수 있는 방법이 없었기에 해당 기능을 불가능으로 마무리 하려고 했지만,
		IP체크를 로그인 서비스를 제공하는 툴 직접 하지않고 외부 API를 호출하여서 체크하면 가능할것 같다는 생각이들어 로그인 모듈중간부분에 API 를 추가하여 IP 체크를 할 수 있도록 하였습니다.
				
	
	2. 당신의 단점은 무엇입니까
		끈기가 너무 강한 것도 단점이라고 생각합니다. 
		이전에 통합계정관리 서비스를 구축할때 사용자의 정보를 엑셀로 내려받는 기능을 개발한 적이있습니다.
		화면단에서 사용한 자바스크립트 라이브러리에서 엑셀파일을 바로 내려받을 수 있는 기능이 있기에 사용하였었는데
		데이터가 2만건이 넘어가면 다운로드가 안되었습니다.
		자바스크립트 라이브러리를 통해서 9만건이 되는 사용자 데이터를 다운로드 하기위해 3일간 라이브러리 파일을 이곳저곳 수정하여서 시도하였지만
		결국 실패하고 백엔드에서 데이터를 처리하여 엑셀다운로드 기능을 개발하여 반나절 만에 끝냈습니다.
		자바스크립트로 다운로드하는것에 대해 좀더 타협하고 2만건 이하는 자바스크립트로 그 이상은 백엔드에서 처리하도록 하였으면 시간소요가 이렇게 크지 않았을것입니다.
