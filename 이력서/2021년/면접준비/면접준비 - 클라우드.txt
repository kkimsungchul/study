2. 뉴타닉스에 대한 설명 
	뉴타닉스는 HCI 기반으로 클라우드 서비스를 제공해주며, 기존 VMware 에서 제공해주는 SDC 에 스토리지 가상화 기능, SDC를 추가한 클라우드 서비스입니다.
	VMware와의 차이점은 DBaaS ,네이티브 오브젝트 스토리지 서비스와 프라이빗 ,퍼블릭 클라우드간의 라이선스 이동성 지원입니다.
	

	- HCI(하이퍼컨버지드 인프라)
		HCI는 하드웨어와 소프트웨어가 함께 제공되는 어플라이언스 형태의 Compute Node
	- 기존 VMware에서 스토리지 가상화 기능 추가 (SDC+SDS) (뉴타닉스와 VMware의 차이를 알고있으면 좋을것같음) 
	
	SDC : Software Defined Compute 소프트웨어 정의 컴퓨터
	SDS : Software Defined Storage 소프트웨어 정의 스토리지

3. 클라우드의 동향 - 향후 퍼블릭의 우세 or 프라이빗의 우세 => 
	
	클라우드 시장은 앞으로 하이브리드 클라우드 쪽으로 더욱 더 성장할 것으로 보입니다.
	정부.공공기관의 클라우드 서비스 도입에 대한 자료예보에 따르면 국내 1만6천여개 시스템중에 13.8% 만 클라우드 서비스를 도입한 것으로 나오며, 향후 28.9%는 클라우드 서비스를 도입할 예정이라고 조사되었습니다.
	또한 2025년까지 정부의 정보시스템들을 100% 클라우드로 전환할 계획이 있다고 합니다.
	이와같이 퍼블릭 클라우드와 프라이빗 클라우드 둘다 성장할 것으로 보이며, 
	엔터프라이즈환경에서는 데이터 보안과 퍼블릭 클라우드의 과금 때문에 프라이빗 클라우드를 사용을 하는 추세지만 , 퍼블릭 클라우드의 유연성이 필요하기때문에 하이브리드 클라우드쪽으로 가야된다고 생각합니다.



4. DevOps / MSA / CI/CD 의 차이점 or 경험 ?
	
	DevOps 경험

		DevOps 툴은 Jira와 Confluence, Gitlab , Jenkins 를 사용했습니다.
		jira 와 Confluence 를 사용하여 개발 일정 및 개발내용을 협의하고 Gitlab을 사용하여 코드 관리를 하였습니다.
		Jenkins 는 운영환경에서는 보안상의 이유로 사용하지 못해 테스트환경에 빌드 및 배포할 때 사용했습니다.

	MSA
		MSA는 직접 사용해본적은 없지만 서비스가 커지면 저킬수록 모놀로틱 형태의 개발에서 발생할수 있는 문제를 보완해줄수 있습니다.
		문제점으로는 부분장애가 전체 서비스의 장애로 이어지는경우, 
		빌드, 테스트, 배포시간이 늘어나는 경우, 서비스를 부분적으로 scale-out 해야하는 경우등이 있습니다.
		
		MSA를 도입하면 전체 서비스 중단 없이 서비스별 배포가 가능하고, 특정 서비스의 확장성이 좋아집니다.
		또한 장애가 전체 서비스로 확장될 가능성도 적습니다.

		다만 서비스간 통신을 사용하기에 서비스간 호출 API가 많아지며
		테스트 / 트랜젝션의 복잡도가 증가합니다.
		또한 데이터가 여러 서비스에  걸쳐 분산되기 때문에 한번에 조회하기가 어렵습니다.

	CI/CD
		현재 맡은 프로젝트에서 운영환경은 보안상의 이슈때문에 사용하지 못하지만 테스트환경에는 Jenkins 사용하여 빌드 및 배포를 하고 있습니다.
		CI/CD 는 지속적 통합과 지속적 제공 또는 지속적 배포입니다.
		CI는 개발자들이 형상관리 툴에 작업한 코드를 빌드 및 테스트를 자동으로 진행해서 통합해주고
		CD는 검증이 끝난 소프트웨어를 배포하는것을 말합니다.

#CMP에 대해 얼마나 알고 있냐
	CMP란?
		다양한 클라우드 시스템이 자체적으로 고유하게 가지고 있는 기능들을 공통된 규격으로 서비스화 하여 제공해고
		클라우드 서비스를 조회하고 신청하는 워크플로우를 표준화 하여 사용자 중심의 클라우드 플랫폼 운영을 가능하게 해줍니다.

		뉴타닉스(Nutanix) 인프라 기반으로 상용화한 최초의 클라우드 플랫폼입니다.

클라우드 서비스에 관심이 있어 공부하고 사용해보다가 CMP에 대해 알게되었습니다.
CMP는 클라우드 매니지먼트 플랫폼으로 다양한 클라우드 시스템에서 제공해주는 고유의 기능들을 표준화 하여 사용자들이 편리하게 클라우드 서비스를 사용할 수 있도록 해준다고 되어있습니다.
CMP를 서비스하는 기업들을 찾아보니 신세계아이앤씨와 SKT , 베스핀글로벌, 이노그리드 등의 업체가 서비스 하는것을 확인하였습니다.
SKT_CMP와 베스핀글로벌의 OpsNow는 비용관리에 중점을 둔 CMP로 VM생성에 대한 부분은 보이지 않았습니다.
이노그리드의 PaaS 기반의 CMP를 서비스 하는것으로 보였습니다.
클라우드 매니지먼트 플랫폼이라고 한다면 가상자원을 생성하고 관리 할수있어야 하는데 그 부분이 명확하게 보이는 CMP는 없었습니다.
신세계아이앤씨의 스파로스 CMP는 조사를 해보아도 엔터프라이즈 환경에 최적화된 클라우드 프로세스를 
뉴타닉스(Nutanix) 인프라 기반 IaaS CMP라는 정보만을 접할수 있었습니다.
뉴타닉스는 클라우드 서비스를 제공해 주는 기업이며, 뉴타닉스 기반이라고 한다면 클라우드의 생성까지 지원할 것이라고 생각합니다.


2022년 행정·공공기관 클라우드컴퓨팅 수요예보 자료에 따르면 현재 공공분야의 시스템은1만6000개 정도입니다.
이 시스템들의 클라우드 이용률 13.3% 이지만 향후 클라우드 도입 예정은 28.9% 라고 합니다.
또한 25년까지 공공기관의 정보시스템을 100% 클라우드로 전환할 계획 입니다.
앞으로 더 많은 시스템들과 기관에서 클라우드 서비스를 사용할 때 스파로스CMP가 앞장 설 수 있도록 하기 위하여 지원하였습니다.

스파로스 CMP 
	1단계 : 뉴타닉스 기반의 프라이빗 클라우드 관리 시스템으로 시작 사용자-관리자 페이지 분리를 통한 사용자는 자원 신청을 하고 관리자가 자원을 생성 
			(관리자는 자원을 직정 생성하지 않고 웹 상에서 API호출을 해서 자동으로 생성) 미터링/빌링 기능 지원을 통한 클라우드 관리 기능 제공 
	2단계 : AWS 개발을 통한 멀티 클라우드 프로비저닝 기능 지원 향후 데이터 통신이 가능한 하이브리드 클라우드 개발 예정 
	3단계 : VMware 개발을 통한 프라이빗 클라우드 최대 밴더 2개 기능 지원 예정 향후 MSP 포탈에 대한 요구사항 반영이 있을 수 있음 
	MSP / CMP에 대한 구분 필요 
		- MSP : CSP에서 제공하는 클라우드를 파트너사가 관리해주는 포탈 
		- CMP : 클라우드를 운영/관리하는 플랫폼 멀티 클라우드, 하이브리드 클라우드의 구분 필요 
		- CSP : cloud service provider , 클라우드 서비스 제공 업체
		- 하이브리드 : 프라이빗-퍼블릭 데이터 연동 구조 
		- 멀티 : 복수개의 클라우드 사용, 하이브리드 클라우드를 여러개 써도 멀티라고 말할 수 있음

	- 프로비저닝(provisioning): 
		사용자의 요구에 맞게 시스템 자원을 할당, 배치, 배포해 두었다가 필요 시 시스템을 즉시 사용할 수 있는 상태로 미리 준비해 두는 것을
