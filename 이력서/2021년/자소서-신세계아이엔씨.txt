# 지원 동기 및 포부 - 700자
저는 개발자로 성장해오면서 고객들과 소통하며 고객중심의 플랫폼 서비스를 개발하는 것을 목표로 생각하고 있습니다.
이러한 저의 목표를 이루기 위해 함께 성장해갈 수 있는 최고의 IT 회사를 찾던 중 신세계아이앤씨의 "MSA 기반의 클라우드 플랫폼(스파로스 CMP) 개발" 채용공고를 보게 되었습니다.
스파로스 CMP는 클라우드 서비스들을 한 번에 통합해서 사용자에게 손쉽고 편리하게 관리할 수 있는 플랫폼으로써 빠르게 변하는 IT업계에서 선택이 아닌 필수가 되어가고 있는 클라우드 서비스에서 꼭 필요다 생각하고, 저의 개발자 목표와 매우 같다고 느끼게 되어 지원하게 되었습니다.
입사 후 목표를 이루기 위해서 첫 번째로는 팀에 필요한 인재가 되도록 하겠습니다.
저의 개발 경험과 지식을 공유하며 팀에 필요로 하는 개발 능력을 키우며 프로젝트를 수행하다 보면 팀에서 필요한 인재가 될 것이라고 확신합니다.
두 번째로는 회사와 고객이 찾는 인재가 되도록 하겠습니다.
업무를 수행함으로써 "김성철"이 회사에 있으면, 어떠한 상황이 발생하여도 안심할 수 있다는 생각을 가질 수 있도록 할 것입니다.
고객의 요구사항에는 귀를 기울이며 서비스에 반영할 수 있는 손쉬운 서비스를 개발하여 유지 보수 및 추가 요구사항에는 신세계아이앤씨의 인력 낭비가 없도록 하여 고객도 만족하고, 신세계아이앤씨에서도 만족하는 프로젝트가 될 수 있도록 할 것입니다.





# 보유기술 및 경험직무 (구체적으로 기술) - 800자
- SpringBoot로 웹서비스 개발, API Document를 토대로 타 시스템과의 연동 개발 / Jenkins 자동배포 / keycloak 모듈개발
현재는 Dev-space 포털 개발 및 유지 보수(개발 툴 관리 포털) 프로젝트에서 PL을 담당하고 있으며, keycloak 모듈 및 SpringBoot로 Jenkins , Gitlab , Jira 등의 툴을 연동하여 하나의 포털에서 툴들의 CRUD를 할 수 있도록 개발하였습니다.
RestAPI를 주로 사용하여 개발하다 보니 연동하는 툴들의 API Document를 자주 찾아보며 API에 대한 이해와 중요성을 느끼며, 추후 유지 보수와 원활한 프런트 개발자와의 소통을 위해 API 개발 시 Confluence 툴에 API 명세서를 공유하는 것을 지속적으로 하고 있습니다.
또한 타 시스템과의 연동이 주 개발이다 보니 포털 DB에 저장된 데이터와 각 툴과의 데이터가 일치하지 않을 경우 오류가 발생하거나 의도치 않은 데이터를 수정하는 등의 오류가 발생할 수 있어 예외 처리 및 롤백 정책의 중요성을 느끼고 개발 시에 발생할 수 있는 예외들에 대한 처리 방법과 롤백에 대해 적극적으로 나서서 개선 및 개발하고 있습니다.

- Spring / JAVA Batch 를 사용하여 웹서비스 개발
이전 프로젝트에서 진행한 통합 사용자 계정관리 솔루션 개발에서는 Spring을 통한 사용자에게 매니저 화면 제공과 Java Batch를 통해서 Tibero-> MSSQL->AD로 데이터 연동 기능을 개발하였습니다.




# 경력기술서 - 700자
- Dev-space 포털 개발 (개발툴관리포털)
KT에서 개발 및 업무에 자주 사용하는 툴(Jenkins , Jira , Gitlab , Sparrow 등)들을 엮어 하나의 포털에서 사용자 인증과 관리할 수 있는 플랫폼입니다.
포털은 SpringBoot로 개발하였으며 툴에서 제공해 주는 API Document를 토대로 연동하여 툴들을 관리할 수 있는 Backend API 개발에 참여하였으며,

SpringBoot로 개발하였으며 Tibero, MSSQL , AD 와 연동하여 사용자의 정보를 관리하는 기능을 개발하였습니다.
사용자 관리(CRUD), 부서 관리(CRUD) 개발 연동 모듈은 회사에 존재하여서 각각의 시스템에 맞도록 커스터마이징을 담당하였습니다.

# 성격 장점 및 보완점 - 300자
책임과 끈기, 문제해결 능력
개발 진행 시 예기치 못한 변수로 업무에 차질이 생기면 책임지고 문제를 해결 할 수 있을때 까지 포기하지 않는 끈기와 해결 방법을 하나로만 생각하지않고 다방면으로 생각하여 문제를 해결 할 수 있도록 합니다.
