# rocky linux에 log파일 용도
	
	- /var/log/maillog
	용도: 메일 관련 로그로, 메일 서버 또는 시스템에서 발생하는 이메일 전송과 관련된 정보를 제공합니다.
	활용: 메일 송수신 문제, 스팸 필터링, 메일 서비스 상태 점검 등에 유용합니다.

	- /var/log/audit/audit.log
	용도: 시스템의 보안 관련 이벤트를 기록하며, SELinux와 같은 보안 이벤트 및 시스템 콜을 추적합니다.
	활용: 보안 사고 분석, 사용자 및 프로세스의 권한 변경 모니터링, 정책 위반 탐지 등에 사용됩니다.

	- /var/log/cron
	용도: 주기적으로 실행되는 크론 작업의 로그를 기록합니다.
	활용: 예약 작업의 실패 또는 성공 여부를 확인하고, 작업 스케줄에 관련된 문제를 파악하는 데 유용합니다.

	- /var/log/dmesg
	용도: 시스템 부팅 시 커널 관련 메시지를 기록합니다.
	활용: 하드웨어 및 커널 관련 문제를 탐지하고 디버깅하는 데 도움을 줍니다.

	- /var/log/yum.log
	용도: 패키지 관리자인 yum을 통해 설치, 업데이트, 삭제된 패키지에 대한 정보를 제공합니다.
	활용: 시스템 소프트웨어 업데이트 내역을 추적하고, 설치된 패키지의 변경 사항을 관리하는 데 유용합니다.

	- /var/log/httpd/access_log 및 /var/log/httpd/error_log
	용도: Apache 웹 서버를 사용하는 경우, 접속 및 오류 로그를 기록합니다.
	활용: 웹 애플리케이션의 성능, 보안 위협(예: 해킹 시도) 모니터링, 트래픽 분석 등에 유용합니다.


	- /var/log/btmp
	용도: 실패한 로그인 시도 기록입니다.
	활용: 비정상적인 로그인 시도, 예를 들어 무차별 대입 공격(brute force) 탐지에 유용합니다.
	확인 방법: lastb 명령어를 통해 접근할 수 있습니다.

	- /var/log/wtmp
	용도: 성공한 로그인 및 로그아웃 기록입니다.
	활용: 사용자 로그인 내역을 추적하고 서버 접근 이력을 확인하는 데 유용합니다.
	확인 방법: last 명령어를 사용하여 확인할 수 있습니다.

	- /var/log/auth.log (또는 /var/log/secure가 이 용도로 쓰임)
	용도: 인증 관련 로그로, 사용자 권한 변경, 인증 시도 등의 보안 관련 이벤트를 기록합니다.
	활용: 사용자 인증 문제를 추적하고, 비정상적인 접근 시도를 감지하는 데 유용합니다.

	- /var/log/boot.log
	용도: 시스템 부팅 과정에서 발생하는 로그입니다.
	활용: 부팅 과정에서의 에러나 문제점을 디버깅하는 데 도움을 줍니다.

	- /var/log/syslog
	용도: 다양한 시스템 서비스의 공통 로그를 기록합니다.
	활용: 모든 시스템 이벤트를 폭넓게 모니터링하고, 특정 문제의 원인을 추적할 때 유용합니다.
	참고: Rocky Linux에서는 syslog가 messages 파일에 통합되어 있는 경우가 많습니다.

	- /var/log/journal/ (Systemd 기반 시스템의 바이너리 로그)
	용도: systemd-journald가 관리하는 로그로, /var/log/messages 이상의 세부적인 정보를 포함할 수 있습니다.
	활용: 바이너리 형식의 로그로 고급 시스템 이벤트를 탐지하고, journalctl 명령어로 필터링하여 조회할 수 있습니다.

	- /var/log/firewalld
	용도: Firewalld(방화벽 데몬)와 관련된 다양한 이벤트와 설정 변경을 기록합니다.
	활용: 방화벽 설정이 제대로 적용되었는지, 특정 포트나 IP 차단 여부를 확인하고 방화벽 규칙 변경 사항을 추적할 수 있습니다.

	- /var/log/firewalld-droppd.log
	용도: 방화벽에서 차단(dropped)된 패킷에 대한 로그를 기록합니다.
	활용: 네트워크에서 차단된 비정상적 접근 시도나 외부 공격을 식별하고 모니터링하는 데 유용합니다.

	- /var/log/hawkey.log
	용도: hawkey와 관련된 로그 파일로, dnf 또는 yum 등의 패키지 관리자가 패키지 설치, 업데이트, 삭제와 같은 작업에서 hawkey를 호출할 때 생성됩니다.
	활용: 패키지 관리 오류, 설치 또는 업데이트 내역을 추적하여 시스템 소프트웨어 관리에 도움을 줍니다.

	- /var/log/spooler
	용도: 스풀러(spooler) 서비스에 대한 로그로, 프린트 작업과 같은 스풀링이 필요한 작업과 관련된 정보를 기록합니다.
	활용: 프린터 연결 문제나 스풀러 작업 실패를 모니터링하고 문제 해결에 활용할 수 있습니다.

	- /var/log/tuned/tuned.log
	용도: tuned 서비스에 의해 작성되는 로그로, tuned는 시스템 성능과 전력 소비를 최적화하기 위해 다양한 프로파일을 관리하는 데몬입니다.
	활용: tuned가 적용한 시스템 최적화 프로파일, 에러 로그, 적용된 설정 변경 사항 등을 확인하여 시스템 성능 튜닝 상태를 점검할 수 있습니다.
